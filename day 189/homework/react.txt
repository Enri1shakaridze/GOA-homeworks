React

1. რა არის React

React-ის მთავარი იდეაა აპლიკაციის დაყოფა პატარა ნაწილებად, რომელსაც კომპონენტები ეწოდება.

მისი დახმარებით შესაძლებელია სწრაფი, მარტივად სამართავი და მასშტაბური ვებ აპლიკაციების შექმნა.


2. Component

Component არის React-ის მთავარი ნაწილი.

კომპონენტი არის კოდის პატარა დამოუკიდებელი ნაწილი, რომელსაც თავისი ფუნქცია აქვს.

მაგალითად:

Header
Navbar
LoginForm
Footer

კომპონენტების გამოყენება ამარტივებს კოდის მართვას და საშუალებას გვაძლევს ერთი კომპონენტი ბევრგან გამოვიყენოთ.


3. JSX

JSX არის JavaScript-ის სინტაქსი, რომელიც საშუალებას გვაძლევს JavaScript-ში HTML-ის მსგავსი კოდი დავწეროთ.

მაგალითი:

function App() {
  return <h1>Hello React</h1>
}

JSX საბოლოოდ გარდაიქმნება ჩვეულებრივ JavaScript კოდად.


4. Virtual DOM

Virtual DOM არის React-ის ერთ-ერთი მთავარი მექანიზმი.

როდესაც მონაცემი იცვლება, React ჯერ ქმნის Virtual DOM-ს და ადარებს ძველ ვერსიას.

შემდეგ ცვლის მხოლოდ იმ ნაწილს, რომელიც შეიცვალა.

ამის გამო React უფრო ეფექტურად მუშაობს.


5. Declarative Programming

React იყენებს დეკლარაციულ მიდგომას.

ეს ნიშნავს, რომ ჩვენ ვწერთ როგორი უნდა იყოს შედეგი და React თვითონ მართავს ცვლილებებს.

მაგალითად:

თუ მომხმარებელი შესულია, აჩვენე პროფილი.

თუ არ არის შესული, აჩვენე შესვლის ფორმა.


6. State

State არის მონაცემი, რომელიც კომპონენტში ინახება და შეიძლება შეიცვალოს.

მაგალითად:

მომხმარებლის ინფორმაცია
რიცხვი
თემის ფერი
ძებნის ტექსტი
ღილაკის მდგომარეობა

State-ის შეცვლისას React ხელახლა არენდერებს კომპონენტს.


7. Props

Props არის მონაცემი, რომელსაც ერთი კომპონენტი გადასცემს მეორე კომპონენტს.

მაგალითად:

App გადასცემს MovieCard-ს:

ფილმის სახელს
სურათს
რეიტინგს

Props მხოლოდ წასაკითხად გამოიყენება და მისი პირდაპირ შეცვლა არ შეიძლება.


8. One Way Data Flow

React-ში მონაცემები მოძრაობს მხოლოდ ერთი მიმართულებით.

მშობელი კომპონენტიდან შვილ კომპონენტში.

ეს ამარტივებს კოდის კონტროლს და შეცდომების პოვნას.


9. Rendering

Rendering არის პროცესი, როცა React კომპონენტის გამოსახულებას ეკრანზე აჩვენებს.

როდესაც მონაცემი იცვლება, React ამოწმებს რა შეიცვალა და საჭირო ნაწილს ანახლებს.


10. render

როდესაც State ან Props იცვლება, კომპონენტი ხელახლა რენდერდება.

ეს არ ნიშნავს, რომ მთელი გვერდი თავიდან იტვირთება.

React მხოლოდ საჭირო ნაწილს ცვლის.


11. Reconciliation

Reconciliation არის პროცესი, როცა React ადარებს ძველ და ახალ Virtual DOM-ს.

ამ პროცესით React პოულობს განსხვავებებს და ანახლებს მხოლოდ შეცვლილ ნაწილებს.


12. Hooks

Hooks არის სპეციალური React ფუნქციები, რომლებიც კომპონენტებს დამატებით შესაძლებლობებს აძლევს.

ყველაზე ცნობილი Hooks:

useState
useEffect
useRef
useMemo
useCallback
useContext
